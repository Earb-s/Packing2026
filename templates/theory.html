Method Overview

The Theory Behind the Calculation

This page explains the full numerical workflow used by the simulator, including equations, assumptions, and data validation rules.

1. Calculation Pipeline

  1. Read and validate PSD data: CSV content is parsed by column names and values, not by file name.
  2. Sort by particle size: Size (mm) is ordered ascending, then cumulative passing is checked for monotonic increase.
  3. Fit Rosin-Rammler curves: Each active PSD is fit to obtain x0 and n.
  4. Build mixed PSD: Curves are blended using volume fractions derived from mass fractions and densities.
  5. Build class fractions: Mixed cumulative passing is converted to differential class fractions.
  6. Compute wall and loose terms: Interaction terms are evaluated for each class pair.
  7. Compute class packing and final packing: Minimum class packing defines no-compaction packing.
  8. Solve true packing under compaction: Compaction levels K = 4, 6.7, 7, 9 are solved via nonlinear root finding.
  9. Generate outputs: Charts and fit summary metrics are rendered on the results page.

2. PSD Fitting Equation

For each active PSD, the model uses a Rosin-Rammler style cumulative function:

y(x) = 100 - 100 exp(-(x / x0)n)

where x is particle size in mm, x0 is characteristic size (mm), and n controls curve shape. The app reports RMSE and R2 for fit quality.

3. Mixing Mass to Volume Fractions

Input mass fractions are converted to volume fractions before blending PSD curves.

mi,norm = mi / sum(mj)
vi = (mi,norm / rhoi) / sum(mj,norm / rhoj)
Ymix(x) = sum(vi * Yi(x))

rho is density and Y(x) is cumulative passing at size x.

4. Class Fractions and Blended Beta

fk = (Acck - Acck-1) / 100
betablend = sum(vi * betai)

fk is the differential fraction in class k.

5. Packing Equations (de Larrard Style)

Wall and loose interaction terms are evaluated between size classes.

Wi = sumj < i (1 - di/dj)1.3 * fj
Li = sumj > i [0.7(1 - dj/di) + 0.3(1 - dj/di)12] * fj
Pi = betablend / (1 - (1 - betablend)Wi - Li)
Pfinal = min(Pi)

6. True Packing Under Compaction

For each target compaction level K, a nonlinear equation is solved:

phi solves: sum((fi / betablend) / ((1 / phi) - (1 / Pfinal))) - K = 0

The reported values k4, k6.7, k7, and k9 are these solved phi values.

7. Output Interpretation

  • Final Packing Density: No-compaction packing from the limiting class.
  • True Packing (K values): Expected packing under increasing compaction effort.
  • Blended Beta: Intrinsic packing indicator of the blended system.
  • PSD Fit Curves: How well each source PSD is represented by the fitted function.

8. CSV Caution and Required Pattern

Upload names can be anything. The parser ignores filename and validates only column names and values.

Column Requirement
Size Numeric, positive particle size values in mm.
Acc from small Numeric cumulative passing from small size, bounded 0 to 100.
Monotonic rule After sorting by Size (mm) ascending, Acc from small must be non-decreasing.
End condition The final cumulative value at the largest size must be 100 (with small numeric tolerance).
